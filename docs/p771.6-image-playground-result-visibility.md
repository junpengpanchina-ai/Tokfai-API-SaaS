# P771.6 — Image Playground result visibility

**Scope:** `apps/web` layout and client state only — no Image API, billing, or ledger changes.

## Problem

After P771.5 two-column layout, the generation result was easy to miss: users clicked
Generate but did not see loading or the finished image.

## Fix

| Behavior | Implementation |
| --- | --- |
| Result card first in right column | Fixed order: Result → Settings → Service docs |
| Stable result footprint | `min-height: 360px` on Result card + dashed inner panel |
| Generate feedback | `resultAttention` ring for ~4s; loading skeleton in Result card |
| Mobile scroll | `scrollIntoView` on Generate start and on complete when off-screen |
| Desktop scroll | No jump when Result is already in viewport; scroll only if top is off-screen |
| Error recovery | Retry button re-submits the form |

## Files

- `apps/web/components/image-playground-toolbench.tsx` — Result states, `focusImagePlaygroundResultPanel`
- `apps/web/app/dashboard/image-playground/image-playground-client.tsx` — refs, attention, scroll hooks
- `apps/web/lib/i18n/messages.ts` — Generation result / loading / retry strings
